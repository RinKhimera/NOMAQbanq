## 2. Conventions à suivre (déjà établies, ne pas réinventer)

- Structure : `features/<domaine>/{schemas.ts, dal.ts, actions.ts}`.
  - **DAL** : `import "server-only"` + self-guard (`requireSession`/`requireRole`
    via `@/lib/auth-guards`, ou `getCurrentSession` de `@/lib/dal`) + React
    `cache()` + colonnes ciblées. Comptes via SQL live (pas de tables d'agrégat).
  - **Actions** : `"use server"` → guard → zod (`safeParse`, fail(message)) →
    écriture → `revalidatePath`. Concurrence par-utilisateur = `db.transaction`
    + `SELECT … .for("update")` (verrou de ligne).
- **Forme « pont » des questions** pour réutiliser les composants quiz partagés
  (`QuestionCard`, etc.) : le DAL renvoie `_id`/`_creationTime`/`images:{url,
  storagePath,order}` ; `correctAnswer`/`explanation`/`references` **présents
  seulement quand autorisé** (admin ou session complétée). Au point de montage,
  `question={q as never}` (le brand `Id` Drizzle = `string`).
- **Écrans** : page = Server Component qui fetch la DAL et passe en props à un
  wrapper client (`*-client.tsx`) ; mutations via Server Actions + `router.refresh`
  (plus de réactivité Convex). Voir `admin-exams-client.tsx`, `exam-details-client.tsx`.
- **Types DAL côté client** : `import type { X } from "@/features/.../dal"`
  (erasé → pas de fuite du module `server-only` dans le bundle client). C'est OK
  et déjà fait partout.
- Langue UI = **français avec accents**. Routes user en français.

## 3. Pièges déjà rencontrés (gagne du temps)

- **ESLint `react-hooks/purity`** : interdit `Date.now()`/`new Date()` argless
  **dans le corps de rendu d'un Server/Client Component**. → sortir l'horloge
  dans un helper au scope module (cf. `isExamClosed` dans
  `examen-blanc/[examId]/page.tsx`).
- **ESLint `react-hooks/set-state-in-effect`** : pas de `setState` synchrone dans
  un `useEffect`. Pour un fetch-par-id, tracker l'id chargé : `useState<{id,q}>` +
  comparer `state?.id === currentId` au lieu de reset synchrone (cf.
  `question-preview-panel.tsx`).
- **Next 16** : `params` est une **Promise** → `const { id } = await params`.
- **`loadQuestionById` est admin-only** (`requireRole(["admin"])`) → ne JAMAIS
  l'appeler depuis un écran étudiant (sinon redirect). `ExamQuestionsModal` a un
  prop `enableDetails` (admin only) pour ça.
- **Prettier réordonne les imports** : lancer `bunx prettier --write <fichiers>`
  AVANT le commit, puis re-`bun run check`.
- **SonarLint** (codes `typescript:Sxxxx` : S3776 complexité, S6759 readonly
  props, S7749 littéraux numériques…) = **IDE-only**, ne casse PAS `bun run check`
  (= `tsc --noEmit && eslint --max-warnings 0`). Les ignorer.
- `submitExamAnswers` a une complexité cognitive élevée (S3776) — c'est attendu
  (anti-triche/budget-temps/pause), ne pas « refactorer pour SonarLint ».

## 3. Conventions (rappel — détail dans le handoff 5.5 §2 + `.claude/rules/data-layer.md`)
- `features/<domaine>/{schemas,dal,actions}.ts` ; DAL `server-only` + `cache()` +
  self-guard (les stats admin = `requireRole(["admin"])` ; étudiant =
  `getCurrentSession`/`requireSession` ; marketing = public sans garde).
- Page = Server Component fetch DAL → props → wrapper client ; mutations via
  Server Actions + `router.refresh()`.
- Hook client qui doit fetcher → action via `useEffect` + `setState` dans `.then`
  (parité loading state, hors `set-state-in-effect`). Cf. `useMarketingStats` (5.6a).
- `react-hooks/purity` : pas de `Date.now()`/`new Date()` argless dans le rendu →
  helper module-scope. `react-hooks/set-state-in-effect` : pas de setState
  synchrone dans un effet.
- Gates par commit : `bun run check` (0) + `bun run build` (0) ; +
  `bun run test:integration` (branche Neon éphémère) **si le DAL/actions change**.
  SonarLint (`typescript:Sxxxx`) = IDE-only, ignorer.
